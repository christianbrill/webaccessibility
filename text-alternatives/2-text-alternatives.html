Text Alternatives

When adding non-text content on your website (images, audio, video, etc.), it is important to always provide a text alternative, so that it can be read by screen readers (SR) such as ChromeVox.

Examples:

  • The <img> element should always be accompanied by an "alt" attribute. The alternative text should be concise and to the point:
  • <img src="dog.png" alt="Image of a Golden Retriever">
  • <input> fields should always have a "name" attribute, which corresponds to the input purpose:
  • <input type="text" name="firstname">
  • CAPTCHAs, themselves being non-text content, must be explained properly and alternatives for different senses must be provided.
  • Non-text content that is purely decorative must be used only for visual formatting or made to be ignored by assistive technology.
